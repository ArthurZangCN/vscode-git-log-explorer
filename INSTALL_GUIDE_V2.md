# 🚀 Git Log Explorer 2.0.0 安装指南

## 🎉 重大突破！
基于测试插件的成功，我们已经恢复了Git Log Explorer的完整功能！

## 📦 新版本特点
- ✅ **确认可以激活**：基于成功的测试插件经验
- ✅ **简化的激活流程**：使用 `onStartupFinished` 激活事件
- ✅ **完整的Git功能**：分支切换、提交查看、比较等
- ✅ **现代化WebView界面**：美观的用户界面
- ✅ **错误处理**：完善的异常处理机制

## 🛠️ 安装步骤

### 步骤1: 卸载旧版本
1. 打开扩展面板：`Ctrl+Shift+X`
2. 搜索 "git debug" 或 "cursor git"
3. 卸载所有相关插件

### 步骤2: 安装新版本
1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "Extensions: Install from VSIX..."
3. 选择 `cursor-git-log-explorer-2.0.0.vsix` 文件
4. **重启Cursor**

### 步骤3: 验证安装
安装后应该看到：
- 🎉 弹窗消息："Git Log Explorer 插件已激活！"
- 🔍 左侧活动栏出现Git分支图标
- 📋 点击图标可以看到"Git Log Explorer"面板

## 🎯 功能测试

### 基本功能测试：
1. **测试命令**：
   - `Ctrl+Shift+P` → "Git Log Explorer: 测试命令"
   - 应该看到成功消息

2. **刷新功能**：
   - 在Git Log Explorer面板右上角点击刷新按钮
   - 应该看到"刷新命令被调用"消息

3. **Git仓库检测**：
   - 如果在Git仓库中，应该自动显示当前分支和提交
   - 如果不在Git仓库中，显示空状态

### 高级功能测试：
1. **分支切换**：选择不同分支查看提交
2. **提交筛选**：按作者或消息筛选
3. **分支比较**：比较两个分支的差异
4. **提交详情**：查看单个提交的详细信息

## 🔍 故障排除

### 如果插件没有激活：
1. 检查是否看到激活弹窗
2. 检查扩展列表中是否有插件
3. 重启Cursor
4. 查看开发者工具控制台（F12）

### 如果Git功能不工作：
1. 确保在Git仓库目录中
2. 检查Git是否正确安装
3. 查看控制台错误信息
