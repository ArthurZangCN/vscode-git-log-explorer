# 📋 插件激活日志查看详细指南

## 🎯 目标
安装 `git-debug-test-1.0.1.vsix` 后，查看插件是否激活以及激活日志。

## 📍 日志查看的4个位置

### 🔥 方法1: 主开发者工具（最重要）
1. **打开开发者工具**：
   - 按 `F12` 键
   - 或者 `Ctrl+Shift+I`
   - 或者菜单：`Help` → `Toggle Developer Tools`

2. **点击Console标签页**
3. **查找这些日志**：
   ```
   🚀🚀🚀 Git Debug Test 插件开始激活... 🚀🚀🚀
   📍 当前工作目录: /path/to/workspace
   📍 插件路径: /path/to/extension
   📍 VSCode版本: x.x.x
   📍 激活时间: 2024-xx-xx...
   ✅✅✅ Git Debug Test 插件激活完成！✅✅✅
   ```

4. **每5秒还会看到**：
   ```
   ⏰ Git Debug Test 插件仍在运行... 2024-xx-xx...
   ```

### 🔧 方法2: 扩展主机开发者工具
1. **打开命令面板**：`Ctrl+Shift+P`
2. **输入**：`Developer: Open Extension Host Developer Tools`
3. **选择该命令**
4. **在新窗口中点击Console标签页**
5. **查找相同的激活日志**

### 📤 方法3: 输出面板
1. **打开输出面板**：
   - 按 `Ctrl+Shift+U`
   - 或者菜单：`View` → `Output`
2. **在右上角下拉框中选择**：
   - `Extension Host`
   - 或 `Log (Extension Host)`
3. **查找激活日志**

### 📦 方法4: 检查插件安装状态
1. **打开扩展面板**：`Ctrl+Shift+X`
2. **搜索**：输入 "git debug"
3. **应该看到**：`Git Debug Test` 插件
4. **确认状态**：已安装且启用

## 🎉 预期的用户界面提示

安装插件后，应该立即看到两个弹窗：
1. **信息消息**：🎉 Git Debug Test 插件已激活！
2. **错误消息**：🔥 这是一个测试错误消息 - 插件确实激活了！

## 🧪 测试命令
1. **打开命令面板**：`Ctrl+Shift+P`
2. **输入**：`Hello World Test`
3. **执行命令**
4. **应该看到**：✅ Hello World 命令工作正常！

## 📁 调试文件
插件还会在其安装目录创建一个 `debug.log` 文件，记录激活时间。

## 🚨 如果什么都看不到

### 检查清单：
- [ ] 插件是否出现在扩展列表中？
- [ ] 是否重启了Cursor？
- [ ] 开发者工具Console是否有任何错误？
- [ ] 扩展主机开发者工具是否有日志？

### 可能的问题：
1. **插件未安装成功**
2. **Cursor扩展系统问题**
3. **WSL环境限制**
4. **权限问题**

## 📊 反馈格式

请按以下格式反馈：

```
✅ 插件安装状态：[是/否] 在扩展列表中看到 "Git Debug Test"
✅ 弹窗消息：[是/否] 看到激活和错误消息
✅ 控制台日志：[是/否] 在F12开发者工具中看到激活日志
✅ 定时日志：[是/否] 每5秒看到运行日志
✅ 命令测试：[是/否] Hello World命令工作
✅ 扩展主机：[是/否] 扩展主机开发者工具中有日志
```
