# Git Log Explorer 插件测试指南

## 测试环境设置

1. 确保在Git仓库中进行测试
2. 按F5启动调试模式
3. 在新的Cursor窗口中打开一个Git项目

## 核心功能测试

### 🔧 问题1修复验证：插件初始化和Git Log显示

**测试步骤：**
1. 启动插件后，检查左侧活动栏是否有Git分支图标
2. 点击图标，应该看到"Git Log Explorer"面板
3. **修复后**：应该立即显示"📋 筛选控制面板"
4. 展开筛选控制面板，应该看到：
   - 🌿 分支/标签: [当前分支名]
   - 👤 作者筛选
   - 💬 消息筛选
   - 分隔线
   - 提交历史列表（自动加载）

**期望结果：**
- ✅ 插件启动后自动显示当前分支的Git日志
- ✅ 不再显示空白面板

### 🎯 问题2修复验证：内嵌筛选控件

**测试步骤：**

#### 分支切换测试
1. 点击"🌿 分支/标签: [分支名]"行
2. **应该弹出QuickPick选择器**（暂时保留，因为分支列表可能很长）
3. 选择不同分支，验证日志是否更新

#### 作者筛选测试
1. 点击"👤 作者筛选"行
2. **应该弹出输入框**（暂时保留，但计划后续改为内嵌输入）
3. 输入作者名称，如"John"
4. 验证筛选结果是否正确

#### 消息筛选测试
1. 点击"💬 消息筛选"行
2. **应该弹出输入框**（暂时保留）
3. 输入关键字，如"fix"
4. 验证筛选结果是否正确

#### 清除筛选测试
1. 设置任意筛选条件后
2. 应该出现"🗑️ 清除所有筛选"选项
3. 点击后验证筛选条件被清除

**期望结果：**
- ✅ 筛选控件直接显示在主边栏中
- ✅ 筛选状态实时显示
- ✅ 可以组合使用多个筛选条件

1. **大仓库测试**：在有很多提交的仓库中测试
2. **筛选性能**：测试作者和消息筛选的响应速度
3. **分支切换**：测试不同分支间的切换速度

## 错误场景测试

1. **非Git目录**：在非Git目录中打开插件
2. **网络问题**：在网络连接不稳定时测试
3. **权限问题**：测试只读仓库的行为

## 兼容性测试

验证以下命令仍然正常工作：
- `gitLogExplorer.refresh`
- `gitLogExplorer.switchBranch`
- `gitLogExplorer.filterAuthor`
- `gitLogExplorer.filterMessage`
- `gitLogExplorer.compareBranches`
- `gitLogExplorer.interactiveRebase`

## 用户体验评估

### ✅ 期望改进
1. **直观性**：筛选控件直接可见，无需寻找
2. **响应性**：操作后立即看到效果

### 📝 注意事项
- 输入框目前仍使用弹出对话框（QuickPick/InputBox）
- 后续版本可考虑使用真正的内嵌输入控件
- 分支选择保留QuickPick，因为可能有很多分支

## 调试技巧

1. **查看控制台**：按F12查看开发者工具
2. **重新加载**：Ctrl+Shift+F5重启调试会话
3. **检查输出**：查看Output面板的Extension Host日志
4. **Git状态**：确保Git仓库状态正常
