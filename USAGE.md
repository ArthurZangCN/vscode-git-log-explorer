# 使用指南

## 快速开始

### 1. 安装和设置

**Linux/macOS 用户:**
```bash
chmod +x setup.sh
./setup.sh
```

**Windows 用户:**
```cmd
setup.bat
```

### 2. 开发和测试

1. 在 Cursor 中打开项目目录
2. 按 `F5` 启动调试模式
3. 在新窗口中打开一个包含Git仓库的项目
4. 在资源管理器面板中找到 "Git Log Explorer"

## 主要功能使用

### 🔄 刷新Git日志
- 点击刷新按钮更新当前分支的提交历史

### 🌿 切换分支/标签
- 点击分支图标
- 选择要查看的分支或标签
- 不会影响工作目录的实际分支

### 🔍 筛选提交
- 点击筛选图标
- 输入作者名称（可选）
- 输入提交信息关键字（可选）

### ⚖️ 比较分支
- 点击比较图标
- 选择起始分支/标签
- 选择结束分支/标签
- 选择是否应用筛选条件

### 📁 查看提交详情
- 点击提交前的箭头展开
- 查看该提交修改的所有文件
- 双击文件名查看差异对比

### 🔀 交互式变基
- 按住 Ctrl 键选择多个提交
- 点击合并图标
- 按照提示进行操作

## 键盘快捷键

- `Ctrl + 点击`: 多选提交
- `双击文件`: 查看文件差异
- `F5`: 启动/重启调试模式

## 故障排除

### 插件不显示
- 确保当前工作目录是Git仓库
- 检查是否正确安装了依赖

### 提交列表为空
- 检查当前分支是否有提交
- 尝试刷新或切换到其他分支

### 文件差异无法显示
- 确保Git命令可以正常执行
- 检查文件是否存在于该提交中

## 开发模式

```bash
# 启动监听模式，自动重新编译
npm run watch

# 手动编译
npm run compile
```

在调试模式下，修改代码后需要重新启动扩展宿主窗口（按 Ctrl+R）。
