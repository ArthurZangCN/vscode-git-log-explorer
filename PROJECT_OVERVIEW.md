# Cursor Git Log Explorer - 项目概览

## 🎯 项目目标

创建一个功能强大的Cursor插件，实现完整的Git日志管理和分支比较功能，提供直观的用户界面和丰富的交互体验。

## ✨ 已实现的功能

### 1. ✅ 侧边栏集成
- 在资源管理器侧边栏显示"Git Log Explorer"面板
- 自动检测Git仓库并显示当前分支日志

### 2. ✅ Git日志显示
- **第一行**: 短提交ID + 作者名称
- **第二行**: 日期 + 提交说明
- 支持展开查看提交的修改文件

### 3. ✅ 分支/标签切换
- 支持切换到任意本地分支、远程分支或标签
- 切换不影响工作目录的实际分支状态
- 实时更新对应的Git日志

### 4. ✅ 文件差异查看
- 双击文件名在右侧打开差异比较视图
- 左侧显示提交前代码，右侧显示提交后代码
- 支持语法高亮和差异标注

### 5. ✅ 智能筛选功能
- **作者筛选**: 按作者名称筛选提交
- **消息筛选**: 按提交说明关键字筛选
- 支持组合筛选条件

### 6. ✅ 分支比较
- 选择两个不同的分支或标签进行比较
- 显示两个分支之间的差异提交
- 可选择是否应用筛选条件

### 7. ✅ 交互式变基界面
- 提供用户友好的交互式变基操作指导
- 支持多选提交进行批量操作
- 类似 `git rebase -i` 的功能体验

### 8. ✅ 完整的Web视图
- 美观的提交差异展示面板
- 支持VSCode主题适配
- 清晰的代码差异高亮显示

## 📁 项目结构

```
cursor-git-log-explorer/
├── src/                          # 源代码目录
│   ├── extension.ts              # 插件入口，注册命令和视图
│   ├── gitService.ts             # Git操作服务，封装所有Git命令
│   └── gitLogProvider.ts         # 树视图数据提供者，处理UI逻辑
├── out/                          # 编译输出目录
│   ├── extension.js              # 编译后的入口文件
│   ├── gitService.js             # 编译后的Git服务
│   └── gitLogProvider.js         # 编译后的视图提供者
├── package.json                  # 插件配置和依赖
├── tsconfig.json                 # TypeScript配置
├── README.md                     # 详细文档
├── USAGE.md                      # 使用指南
├── setup.sh                      # Linux/macOS安装脚本
├── setup.bat                     # Windows安装脚本
└── package.sh                    # 打包脚本
```

## 🛠️ 技术栈

- **语言**: TypeScript
- **框架**: VS Code Extension API
- **Git操作**: simple-git 库
- **UI组件**: VS Code TreeView、WebView
- **构建工具**: TypeScript编译器

## 🚀 核心技术特性

### 1. 模块化架构
- **GitService**: 纯粹的Git操作层，无UI依赖
- **GitLogProvider**: UI数据提供层，处理树视图逻辑
- **Extension**: 入口层，负责注册和协调

### 2. 异步操作处理
- 所有Git操作都是异步的，不阻塞UI
- 完善的错误处理和用户提示
- 优雅的加载状态管理

### 3. 虚拟文档提供者
- 实现自定义URI方案进行文件差异比较
- 动态生成临时文档内容
- 无需创建实际临时文件

### 4. 主题适配
- 完全支持VSCode/Cursor的明暗主题
- 使用CSS变量自动适配颜色方案
- 保持与编辑器一致的视觉体验

## 📋 安装和使用流程

1. **开发环境设置**
   ```bash
   ./setup.sh        # Linux/macOS
   setup.bat         # Windows
   ```

2. **开发调试**
   ```bash
   npm run watch     # 自动编译模式
   # 在Cursor中按F5启动调试
   ```

3. **生产环境打包**
   ```bash
   ./package.sh      # 生成.vsix安装包
   ```

## 🎮 用户交互设计

### 工具栏按钮
- 🔄 **刷新**: 重新加载Git日志
- 🌿 **分支**: 切换分支或标签
- 🔍 **筛选**: 设置筛选条件
- ⚖️ **比较**: 比较两个分支
- 🔀 **变基**: 交互式变基操作

### 交互方式
- **单击展开**: 查看提交文件列表
- **双击文件**: 打开文件差异视图
- **Ctrl+多选**: 选择多个提交进行操作
- **右键菜单**: 快速访问常用功能

## 🔧 扩展能力

插件设计具有良好的扩展性：

1. **新增Git操作**: 在GitService中添加新方法
2. **UI组件扩展**: 在GitLogProvider中添加新的树节点类型
3. **命令扩展**: 在package.json中注册新命令
4. **视图扩展**: 支持添加新的视图面板

## 📊 性能优化

- **懒加载**: 提交文件列表按需加载
- **分页加载**: 限制单次加载的提交数量
- **缓存机制**: 缓存分支和标签信息
- **内存管理**: 及时清理WebView资源

## 🛡️ 错误处理

- **Git操作失败**: 显示用户友好的错误信息
- **无Git仓库**: 优雅地处理非Git目录
- **网络问题**: 处理远程操作超时
- **权限问题**: 处理文件访问权限错误
