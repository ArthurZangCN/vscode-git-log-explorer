# 分支比较功能使用指南

## 🎯 功能概述

Git Log Explorer v2.3.0 引入了全新的分支比较功能，解决了remote-ssh环境的兼容性问题，现在可以在代码编辑器窗口中直观地查看两个分支或标签之间的差异。

## 🚀 如何使用

### 方法1: 通过侧边栏界面
1. 打开Git Log Explorer侧边栏
2. 点击"📋 筛选控制面板"展开
3. 点击"⚖️ 比较分支/标签"
4. 在弹出的选择框中依次选择源分支和目标分支
5. 比较结果将在新的编辑器窗口中显示

### 方法2: 通过命令面板
1. 按 `Ctrl+Shift+P` (Windows/Linux) 或 `Cmd+Shift+P` (Mac)
2. 输入 "Git Log Explorer: 测试分支比较"
3. 选择要比较的两个分支或标签
4. 查看比较结果

## 📊 比较结果说明

比较结果包含以下信息：

### 统计信息
- **仅在源分支中**: 只存在于第一个分支的提交数量
- **仅在目标分支中**: 只存在于第二个分支的提交数量  
- **提交说明不同**: 相同hash但提交消息不同的记录数量

### 详细差异
- **🔴 仅在源分支中的提交**: 显示完整commit ID、作者和提交消息
- **🟢 仅在目标分支中的提交**: 显示完整commit ID、作者和提交消息
- **🔄 提交说明不同的记录**: 显示两个分支中不同的提交消息

## ✨ 特色功能

### 智能过滤
- 自动隐藏相同的提交记录
- 只显示有实际差异的内容
- 减少信息噪音，聚焦重要差异

### 完整信息显示
- 显示完整的commit ID（不是缩短版本）
- 包含作者信息和时间戳
- 支持git-commit语法高亮

### 用户友好
- 清晰的分类和图标标识
- 易于理解的统计信息
- 在独立编辑器窗口中显示，便于查看和保存

## 🔧 技术特点

- **兼容性**: 解决了remote-ssh环境的面板视图问题
- **性能**: 高效的差异算法，支持大型仓库
- **可靠性**: 完善的错误处理和用户反馈
- **扩展性**: 为未来功能扩展预留接口

## 💡 使用技巧

1. **快速测试**: 使用测试命令快速验证功能
2. **保存结果**: 比较结果在编辑器中，可以保存为文件
3. **语法高亮**: 结果使用git-commit格式，支持语法高亮
4. **并排查看**: 使用"在侧边打开"功能并排查看多个比较结果

## 🐛 故障排除

如果遇到问题：
1. 确保当前目录是Git仓库
2. 检查分支或标签是否存在
3. 查看VSCode开发者控制台的错误信息
4. 尝试重新编译插件：`npm run compile`
